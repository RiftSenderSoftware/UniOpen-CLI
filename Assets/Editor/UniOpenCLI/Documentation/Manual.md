# UniOpenCLI — руководство

## Как это работает

Пакет состоит из трёх частей:

1. **Конфигурация** (`CliCommandConfig`) — список команд хранится в `EditorPrefs` под ключом `UniOpenCLI.Commands` в виде JSON. Настройки локальны для машины разработчика: каждый член команды настраивает свой набор CLI, в репозиторий ничего не попадает.

2. **Кодогенерация** (`CliMenuGenerator`) — атрибут `[MenuItem]` в Unity статический, поэтому пункты меню нельзя создать в рантайме из списка. Вместо этого при сохранении настроек генерируется файл `Generated/CliContextMenuGenerated.cs` с одним пунктом меню на каждую команду. При первом запуске (файла ещё нет) генерация происходит автоматически через `[InitializeOnLoad]`.

3. **Запуск** (`CliContextMenu`) — определяет директорию выбранного ассета и запускает `powershell.exe -NoExit` с переходом в неё и выполнением команды. Перед запуском PATH пересобирается из реестра (`Machine` + `User`), потому что Unity передаёт дочерним процессам PATH на момент своего старта — без этого CLI, установленные при открытом редакторе, не находились бы.

## Окно настроек

**Tools → UniOpenCLI**

| Поле / кнопка | Назначение |
| --- | --- |
| Enabled | Выключенная команда остаётся в списке, но её пункт меню неактивен |
| Name | Текст пункта меню (символ `/` заменяется на `-`, дубликаты получают суффикс) |
| Command | Команда, выполняемая в открытом терминале (например, `claude` или `claude --resume`) |
| Add CLI | Добавить новую команду |
| Save | Сохранить в EditorPrefs и перегенерировать меню |
| Discard Changes | Откатить несохранённые правки |
| Reset to Defaults | Вернуть список по умолчанию (требует Save) |

Индикатор в шапке окна показывает состояние: зелёный «Saved» или оранжевый «Unsaved changes» (плюс `*` в заголовке вкладки).

## Выбор рабочей директории

- Выделена папка — терминал откроется в ней.
- Выделен файл — в папке этого файла.
- Ничего не выделено — в корне проекта.

## Ограничения

- Только Windows: терминал запускается через `powershell.exe`. Для macOS/Linux потребуется адаптация `CliContextMenu.Open`.
- Пункты меню обновляются после рекомпиляции — при сохранении настроек Unity на секунду задумается, это нормально.
- Файл в `Generated/` редактировать не нужно: он перезаписывается при каждом сохранении настроек.

## FAQ

**Можно ли запускать не CLI-агентов, а произвольные команды?**
Да, поле Command — это любая строка, выполняемая в PowerShell: `git status`, `npm run dev` и т.д.

**Почему меню не появилось после копирования пакета?**
Дождитесь компиляции. Если файла `Generated/CliContextMenuGenerated.cs` нет, он создастся автоматически при загрузке редактора.

**Как сбросить настройки полностью?**
Кнопка Reset to Defaults + Save, либо удалите ключ `UniOpenCLI.Commands` из EditorPrefs.
